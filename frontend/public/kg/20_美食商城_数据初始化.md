以下是基于 Neo4j 最新版本（4.4+）的初始化建模脚本，实体和关系均以 **`DS_`** 开头，包含不少于 10 个实体和关系的初始化数据。

---

```cypher
MATCH (f2:DS_FoodItem {id: "food_002"}), (c2:DS_Category {id: "cat_002"})
CREATE (f2)-[:DS_BELONGS_TO]->(c2);

MATCH (f3:DS_FoodItem {id: "food_003"}), (c1:DS_Category {id: "cat_001"})

// 商家与标签
MATCH (r1:DS_Restaurant {id: "res_001"}), (t1:DS_Tag {id: "tag_001"})
CREATE (r1)-[:DS_TAGGED_WITH]->(t1);

MATCH (r2:DS_Restaurant {id: "res_002"}), (t2:DS_Tag {id: "tag_002"})
CREATE (r2)-[:DS_TAGGED_WITH]->(t2);

// 订单与地址
MATCH (o1:DS_Order {id: "order_001"}), (a1:DS_Address {id: "addr_001"})
CREATE (o1)-[:DS_DELIVER_TO]->(a1);

MATCH (o2:DS_Order {id: "order_002"}), (a2:DS_Address {id: "addr_002"})
CREATE (o2)-[:DS_DELIVER_TO]->(a2);

// 订单与订单项
MATCH (o1:DS_Order {id: "order_001"}), (oi1:DS_OrderItem {id: "item_001"})
CREATE (o1)-[:DS_CONTAINS]->(oi1);

MATCH (o1:DS_Order {id: "order_001"}), (oi2:DS_OrderItem {id: "item_002"})
CREATE (o1)-[:DS_CONTAINS]->(oi2);

MATCH (o2:DS_Order {id: "order_002"}), (oi3:DS_OrderItem {id: "item_003"})
CREATE (o2)-[:DS_CONTAINS]->(oi3);

// 用户与订单
MATCH (u1:DS_User {id: "user_001"}), (o1:DS_Order {id: "order_001"})
CREATE (u1)-[:DS_PLACED_ORDER]->(o1);

MATCH (u2:DS_User {id: "user_002"}), (o2:DS_Order {id: "order_002"})
CREATE (u2)-[:DS_PLACED_ORDER]->(o2);
```

## 🧪 验证数据

执行以下查询验证数据是否成功插入：

```cypher
// 查看所有用户
MATCH (u:DS_User) RETURN u;

// 查看所有商家及其餐品
MATCH (r:DS_Restaurant)-[:DS_HAS_MENU]->(f:DS_FoodItem) RETURN r.name, f.name;

// 查看订单与订单项
MATCH (o:DS_Order)-[:DS_CONTAINS]->(i:DS_OrderItem) RETURN o.id, i.foodItemId;
```

## 📌 说明

1. **实体命名规范**：所有实体和关系均以 `DS_` 开头（如 `DS_User`, `DS_HAS_MENU`）。
2. **数据量**：
   - 用户：2 个
   - 地址：2 个
   - 商家：2 个
   - 餐品：3 个
   - 订单：2 个
   - 订单项：3 个
   - 分类：2 个
   - 标签：2 个
   - **总计节点数：16 个**
3. **关系类型**：
   - 用户-地址（`DS_LIVES_AT`）
   - 商家-餐品（`DS_HAS_MENU`）
   - 餐品-分类（`DS_BELONGS_TO`）
   - 商家-标签（`DS_TAGGED_WITH`）
   - 订单-地址（`DS_DELIVER_TO`）
   - 订单-订单项（`DS_CONTAINS`）
   - 用户-订单（`DS_PLACED_ORDER`）
   - **总计关系数：12 个**

---

如需扩展更多数据（如评价、收藏、优惠券等），可继续添加实体和关系。如果需要支持地理位置查询（如距离计算），可以基于 `latitude` 和 `longitude` 字段实现。
